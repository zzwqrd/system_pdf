# 🗄️ قاعدة البيانات والترحيل (Database & Migrations)

مشروع **GymSystem** يستخدم محرك **SQLite** المحلي لإدارة كافة بيانات العملاء، المخزون، وعروض الأسعار. تم بناء نظام متقدم للتعامل مع قاعدة البيانات يضمن استقرار البرنامج وتوافقه مع التحديثات.

---

## 📊 مخطط قاعدة البيانات (Database Schema)

تتألف قاعدة البيانات من عدة جداول مرتبطة بعلاقات مفاتيح أجنبية (Foreign Keys):

1.  **جدول الورقات (Quotations):** الخصائص العامة لعروض الأسعار (العميل، التاريخ، الإجمالي).
2.  **جدول البنود (Quotation Items):** التفاصيل الفرعية لكل بند مالي في العرض.
3.  **جدول المواصفات (Quotation Specs):** تصف المزايا الفنية للعرض المرتبط.
4.  **جدول الشروط والأحكام (Quotation Terms):** لتخزين الشروط القانونية المخصصة.
5.  **جدول المخزون (Inventory):** لمراقبة كميات الأدوات والقطع ومتابعة التغييرات التاريخية.
6.  **جدول المستخدمين والأدوار (Users, Roles):** لتأمين الوصول ومنع العمليات غير المصرح بها.

---

## ⚙️ الترحيل المتقدم (Advanced Migrations)

يتفرد البرنامج بنظام ترحيل (Migration Framework) مرن في `lib/core/database/migrations/migration_manager.dart`:

*   **الكشف التلقائي:** يقوم البرنامج بفحص إصدار قاعدة البيانات الحالية ومقارنتها مع الإصدار المستهدف.
*   **التشغيل المتسلسل:** في حال كان الإصدار v10 والمستهدف v16، سيقوم البرنامج بتشغيل جميع الترحيلات من v11 إلى v16 بالترتيب لضمان سلامة البيانات.
*   **المهمة:** كل ترحيل يحتوي على كود SQL أو Dart مخصص لإضافة حقول أو جداول جديدة.

---

## 🌱 نظام بذر البيانات (Seeders)

لضمان سهولة الاختبار والعروض التقديمية، تم دمج نظام **Seeders**:
*   يقوم بإدراج بيانات تجريبية (Sample Data) عند تثبيت التطبيق لأول مرة أو تحديثه.
*   **مثال:** `QuotationSampleSeeder` يقوم بإدراج عروض أسعار نموذجية ببيانات حقيقية (مثل شركة الدقهلية بولتدي) وأدوات صيانة.

---

## 🔒 حماية وسلامة البيانات (Data Integrity)

*   **القيود (Constraints):** استخدام `UNIQUE` لمنع تكرار أرقام العروض و `NOT NULL` للحقول الإجبارية.
*   **المعاملات (Transactions):** يتم تنفيذ العمليات المعقدة ككتلة واحدة؛ إما أن تنجح بالكامل أو تفشل بالكامل (Atomic Operations)، مما يمنع حدوث "نصف بيانات" (Corrupted Data).

---

## 🔍 التعامل مع الاستعلامات المعقدة
*   يتم استخدام `db.rawQuery` للاستعلامات المعقدة جداً، بينما تستخدم العمليات العادية `db.query` و `db.insert` بذكاء باستخدام الـ (Maps).
*   يتم تحويل كائنات الـ Dart إلى Maps والعكس باستخدام موديلات متخصصة في طبقة البيانات.
